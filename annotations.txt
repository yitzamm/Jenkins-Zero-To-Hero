What a Controller is (in Kubernetes)

A controller is a program that constantly watches the cluster and makes sure the actual state matches the desired state. Examples of built-in controllers:

Deployment controller → manages pods
ReplicaSet controller → ensures number of pods
Node controller → watches nodes
Job controller → runs batch jobs

So a controller is basically a Kubernetes automation loop.

What is an operator?

An Operator is a special type of controller designed to manage complex applications. Instead of just managing pods, it manages entire systems like:

PostgreSQL
Prometheus
ArgoCD
Redis
Kafka

An operator knows how to run and maintain the app automatically.

Why Operators are recommended?

Without operators, installing a complex system requires many manual steps.
Important concept: Operators are controllers, but they are controllers specialized for apps.

Other common operators:
Prometheus Operator → installs monitoring stack
ArgoCD Operator → installs GitOps platform
Elastic Operator → installs Elasticsearch
MongoDB Operator → manages Mongo clusters

JENKINSFILE

Stage = Build and Test

On this stage we're cding to the location where the pom.xml file lives. This file contains the dependencies required to run the app. Once there, mvn would download the dependencies, help build the application and generate the artifact (.jar file).

Stage = Static Code Analysis

On this stage we're running mvn sonar:sonar using the application path. The mvn sonar:sonar is a mvn target just as mvn clean package.
